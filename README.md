# TaskFlow

## 1. Overview

TaskFlow is a minimal project/task management app with:

- Go REST API (`chi`, `pgx`, JWT auth, structured JSON errors)
- PostgreSQL (SQL migrations + idempotent seed data)
- React + TypeScript frontend (`React Router`, `TanStack Query`, `MUI`)
- Docker Compose local stack (db + backend + frontend)

Core flows:

- register/login with JWT
- project list/create/detail
- task list/filter/pagination
- task create/edit + optimistic status updates
- project stats by status and assignee

## 2. Architecture Decisions

- Modular monolith backend:
  - Auth, Projects, and Tasks are separated into repository/service/handler modules.
  - Shared platform utilities handle config, middleware, error mapping, and HTTP helpers.

- SQL-first data model:
  - Handwritten migrations and indexes for explicit schema control.
  - No ORM/autogenerated schema layer.

- Integration confidence over mock-only testing:
  - Integration tests run real HTTP handlers against PostgreSQL.

- Minimal but functional frontend:
  - Prioritizes reviewer workflows and backend contract verification over visual complexity.

- Container startup automation:
  - Backend container waits for DB, runs migrations, runs seed (env controlled), then starts API.

## 3. Running Locally

Prerequisites:

- Docker Desktop (or Docker Engine + Compose plugin)

Run:

```bash
git clone https://github.com/abhinavmaity/taskflow.git
cd taskflow
cp .env.example .env
docker compose up --build
```

URLs:

- Frontend: [http://localhost:3000](http://localhost:3000)
- API: [http://localhost:8080](http://localhost:8080)
- Health: [http://localhost:8080/health](http://localhost:8080/health)

Notes:

- `docker compose up --build` is enough from a fresh clone.
- No manual migration/seed step is required for the default flow.

## 4. Running Migrations

Migration behavior:

- Migrations run automatically at backend container startup via `/app/migrate`.
- Applied versions are tracked in `schema_migrations`.

Optional manual commands:

```bash
# Run migrations using local Go toolchain
make backend-migrate

# Run seed script using local Go toolchain
make backend-seed
```

Docker manual run (inside backend container):

```bash
docker compose exec backend /app/migrate
docker compose exec backend /app/seed
```

## 6. API Reference

Auth:

- `POST /auth/register`
- `POST /auth/login`

Projects:

- `GET /projects?page=&limit=`
- `POST /projects`
- `GET /projects/:id`
- `PATCH /projects/:id`
- `DELETE /projects/:id`
- `GET /projects/:id/stats`

Tasks:

- `GET /projects/:id/tasks?status=&assignee=&page=&limit=`
- `POST /projects/:id/tasks`
- `PATCH /tasks/:id`
- `DELETE /tasks/:id`

Example auth request:

```bash
curl -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}'
```

Example protected request:

```bash
curl "http://localhost:8080/projects?page=1&limit=10" \
  -H "Authorization: Bearer <JWT_TOKEN>"
```

Error model:

- `400`: `{ "error": "validation failed", "fields": { ... } }`
- `401`: `{ "error": "unauthorized" }`
- `403`: `{ "error": "forbidden" }`
- `404`: `{ "error": "not found" }`
